# CLAUDE.md 学习档案模板

以下是学习目录 CLAUDE.md 的标准结构。根据实际情况填充，不要照搬空白模板给学习者。

---

```markdown
# [学习室名称]

> [一句话描述这个学习室的用途]

---

## 学习者画像

| 维度 | 内容 |
|------|------|
| **原岗位/技术栈** | [例：Java 后端 3 年 / Python 数据工程 5 年] |
| **编程年限** | [例：3年] |
| **对目标技术了解度** | [例：零基础 / vibe coding 写过但不理解] |
| **学习目标** | [例：能独立开发 RN 新功能] |
| **学习偏好** | [例：喜欢类比式学习，先整体后细节] |

---

## 当前学习：[技术名]（[项目名]）

**工程路径**: `[/path/to/project/]`
**技术栈概述**: [例：React Native + TypeScript + Zustand + React Query]
**开始日期**: [YYYY-MM-DD]

### 知识档案

#### 已了解
- [已掌握的知识点，带简要说明]

#### 进行中
- [正在学习的知识点] ← **当前**

#### 待深入
- [后续要学的知识点]

#### 基础水平评估
- [对基础技能的评估，例：JS/TS语法基本了解，函数式编程有初步认知]

### 认知笔记

> 学习者自己建立的理解方式，比标准定义更有助于记忆

- "[学习者的类比或理解]" — 来自 Level X，[日期]

### 学习路线

#### Level 1: [主题]（[一句话目标]）
> 目标：[具体目标描述]

- [x] [已完成的知识点]
- [ ] [待完成的知识点] ← **当前进度**
- [ ] [后续知识点]

**对应项目文件**: `[src/xxx/]`

#### Level 2: [主题]（[一句话目标]）
> 目标：[具体目标描述]

- [ ] [知识点]
- [ ] [知识点]

**对应项目文件**: `[src/xxx/]`

[... 更多 Level ...]

### 进度日志

| 日期 | 完成内容 | 备注 |
|------|---------|------|
| [YYYY-MM-DD] | [完成的知识点] | [特殊备注] |

---

## 已完成 / 暂停的学习

### [技术名]（[项目名]）— [状态：已完成/暂停]
**完成日期**: [YYYY-MM-DD]
**掌握程度**: [能读懂 / 能改 / 能独立开发]
**收获概要**: [一段话总结]
```

---

## 使用说明

1. **初次使用**：通过对话采集画像信息后，填充模板生成 CLAUDE.md
2. **每次教学**：更新知识档案、认知笔记和进度日志
3. **切换项目**：新增一个「当前学习」区块，旧的移到「已完成/暂停」
4. **跨项目知识**：通用概念（如 JS 语法、HTTP 协议）标记在画像区的基础水平中，跨项目生效
5. **不要在工程目录的 CLAUDE.md 写学习内容** — 工程的 CLAUDE.md 只放项目规范
