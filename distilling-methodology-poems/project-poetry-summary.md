# Skills 仓库方法论歪诗

> 用韦青方法论做诗的方式，将本 Skills 仓库的核心理念浓缩为可背诵的口诀。

---

## 一、七字歪诗——Skills 生态总纲

```
寻建链用聚养生
```

### 逐字拆解

- **寻**：寻找（find-skill）——从生态中发现已有的 Skill，不重复造轮子
- **建**：构建（local-skills-creator）——找不到就自己创建，填补空白
- **链**：链接（linking-skills）——用软链接跨平台共享，Claude/Codex/Cursor/Gemini 通用
- **用**：使用——在实际项目中调用 Skill，让 AI 真正干活
- **聚**：聚合——将散落各处的优质 Skill 汇聚到一个仓库索引
- **养**：养护（git submodule update）——持续更新维护，子模块常拉常新
- **生**：生态——最终形成自我生长的技能生态，1200+ Skill 互相补充

### 结构

前四字"寻建链用"是**个人维度**（从找到用），后三字"聚养生"是**生态维度**（从聚合到生长）。个人用 Skill 的闭环驱动生态的繁荣。

### 使用场景

每次新建项目、接手新任务、或想扩充 Skill 库时默念一遍：先寻再建，链好就用，聚起来养起来，生态自然生。

## 二、四句歪诗——技能域全景

```
开码建测部署完    ← 开发工具链（从编码到部署的完整流程）
文图声影创意宽    ← 内容创作域（文字、图片、音频、视频）
引流转化留存算    ← 营销增长链（获客→转化→留存→分析）
千技汇聚指间弹    ← 1200+ Skill 触手可及
```

### 逐句拆解

**第一句：开码建测部署完**
- 开 = 开发立项（brainstorming, writing-plans）
- 码 = 编码实现（executing-plans, TDD）
- 建 = 构建打包（maven-operating）
- 测 = 测试验证（verification-before-completion, webapp-testing）
- 部署 = 发布上线（docker-bluegreen-deploy, deploy-to-vercel）
- 完 = 完成闭环（finishing-a-development-branch）
- 属于"工程"维度

**第二句：文图声影创意宽**
- 文 = 文案写作（copywriting, baoyu-format-markdown）
- 图 = 图片制作（baoyu-xhs-images, baoyu-infographic, canvas-design）
- 声 = 音频播客（xiaoyuzhou-downloader, podcast-studio）
- 影 = 视频制作（video-downloader, NanoBanana-PPT 转场视频）
- 创意宽 = 创作手段丰富（漫画、PPT、封面图、GIF...）
- 属于"创作"维度

**第三句：引流转化留存算**
- 引流 = 获取流量（SEO, paid-ads, content-strategy, cold-email）
- 转化 = 转化用户（page-cro, signup-flow-cro, paywall-upgrade-cro）
- 留存 = 留住用户（onboarding-cro, email-sequence, referral-program）
- 算 = 数据分析（analytics-tracking, ab-test-setup）
- 属于"增长"维度

**第四句：千技汇聚指间弹**
- 千技 = 1200+ 个 Skill
- 汇聚 = 9 个来源仓库聚合
- 指间弹 = 一条命令即可安装使用
- 属于"愿景"维度

## 三、否定递进——Skill 使用心法

```
有 Skill ≠ 会用 Skill
会用 ≠ 用对场景
用对 ≠ 形成习惯
习惯 ≠ 生态贡献
```

- 第一层：装了不等于会用，要读 SKILL.md 理解触发条件
- 第二层：会用不等于用对，要在正确场景触发正确 Skill
- 第三层：用对不等于习惯，要融入日常工作流
- 第四层：习惯不等于贡献，最终要反哺生态——创建新 Skill、改进旧 Skill

## 四、三字诀——Skill 开发法

**核心三字**：探 → 炼 → 享

- **探**：探索需求（在实际工作中发现痛点，记录重复劳动）
- **炼**：提炼为 Skill（用 local-skills-creator 或 skill-creator 标准化封装）
- **享**：共享生态（提交到仓库，让团队和社区受益）

**操作纲领**（三句口诀）：
```
探痛点    ← 从重复操作中发现值得自动化的环节
炼成技    ← 写成标准 SKILL.md，附示例和触发词
享社区    ← 推到仓库、加子模块、更新索引
```

## 五、递进句式——韦青站桩口诀映射

将韦青的站桩口诀映射到 Skill 使用哲学：

```
手动不如半自动
半自动不如全自动
全自动不如智能化
智能之动
乃 AI 生生不息之动
```

- 手动操作（手写部署脚本）→ 半自动（有模板但要改）→ 全自动（一条 Skill 搞定）→ 智能化（AI 根据场景自动选择 Skill）
- 最终愿景：AI Agent 自己判断什么时候该用什么 Skill，人只需描述意图。

## 风格说明

以上口诀遵循韦青方法论做诗的核心原则：
- **一字一义**：每个字对应一个具体概念
- **有结构**：字与字之间有递进或并列关系
- **可默念**：适合在工作间隙反复玩味
- **谦逊命名**：自称"歪诗""口诀"，不自封经典
- **可拆解**：每个字都能展开为一段完整解释
